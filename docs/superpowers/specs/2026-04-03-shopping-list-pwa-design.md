# Group Shopping List PWA — Design Spec

## Overview

A shared group shopping list PWA for a small household (2-5 people). Real-time sync via Supabase so everyone sees updates instantly. Installable on iPhone home screen for a native app feel. Hebrew-first with bilingual support.

## Users & Auth

- **Target**: small group — household, friends, roommates (2-5 people)
- **Adults**: magic link (email) authentication via Supabase Auth
- **Kids/guests**: shareable invite link — no email needed, joins the group directly
- **Invite link**: generated by any group member, contains a unique token. Opening the link creates an anonymous Supabase session and adds the user to the group as a 'member'. Token stored in `group_invites` table with expiry.
- **Onboarding**: auto-create a group on first login. No explicit group management UI in MVP. First user gets a group created automatically (role: 'admin'); subsequent users join via invite link (role: 'member').

## Tech Stack

- **React 18+** (Vite) + TypeScript
- **Supabase** — auth, real-time Postgres, row-level security, storage
- **Tailwind CSS** — styling
- **vite-plugin-pwa** — service worker, app shell caching, installability
- **Vercel** — deployment
- System font stack for performance

## Architecture

### Approach: Group Implicit

Full Supabase schema (groups, members, lists, items, item_images) with RLS from day 1, but group is auto-created on first login. No settings/group-management screen in MVP. This avoids painful schema migrations later while keeping onboarding frictionless.

### Real-Time

Supabase realtime subscriptions on `items` and `lists` tables. When someone checks an item, adds a new one, or creates a list, all connected clients update immediately.

### Offline (MVP)

PWA installable with app shell caching only. Full offline editing with change queue and conflict resolution is deferred to a later phase.

### Language & Layout

- Bilingual from start: Hebrew (default) + English
- i18n infrastructure with locale files from day 1
- RTL layout as default (`dir="rtl"`, `lang="he"`)

## Data Model

### Tables

`**groups`**

- id (uuid, PK)
- name (text)
- created_at (timestamptz)

`**group_members**`

- id (uuid, PK)
- group_id (uuid, FK → groups)
- user_id (uuid, FK → auth.users)
- role (text: 'admin' | 'member')
- display_name (text)
- avatar_url (text, nullable — Supabase Storage path)

`**lists**`

- id (uuid, PK)
- group_id (uuid, FK → groups)
- name (text)
- icon (text — emoji)
- created_by (uuid, FK → auth.users)
- created_at (timestamptz)

`**items**`

- id (uuid, PK)
- list_id (uuid, FK → lists)
- name (text)
- quantity (integer, default 1)
- unit (text, nullable)
- category (text, default 'other')
- checked (boolean, default false)
- checked_by (uuid, nullable, FK → auth.users)
- added_by (uuid, FK → auth.users)
- note (text, nullable)
- position (integer)
- created_at (timestamptz)
- updated_at (timestamptz)

`**item_images**`

- id (uuid, PK)
- item_id (uuid, FK → items)
- storage_path (text)
- created_at (timestamptz)

`**group_invites**`

- id (uuid, PK)
- group_id (uuid, FK → groups)
- token (text, unique)
- created_by (uuid, FK → auth.users)
- expires_at (timestamptz)
- created_at (timestamptz)

### Row-Level Security

All tables scoped to group_id via group_members join. Users can only read/write data belonging to their group.

### Categories (predefined)

ירקות ופירות (Produce), מוצרי חלב (Dairy), בשר ודגים (Meat & Fish), מאפים (Bakery), קפואים (Frozen), שימורים (Canned Goods), חטיפים (Snacks), משק בית (Household & Cleaning), אחר (Other)

## Image Storage

- **Supabase Storage** bucket: `item-images`
- Path structure: `{group_id}/{item_id}/{filename}`
- RLS on the bucket scoped to group members
- **Capture**: camera + gallery via `<input type="file" accept="image/*" capture="environment">`
- **Processing**: client-side resize/compress before upload (max 1200px wide, JPEG 80% quality)
- **Deletion**: removing an image deletes both the `item_images` row and the storage object
- Free tier: 1GB storage — compressed images (~100-200KB each) = thousands of photos

## Screens

### 1. Login

- Centered layout with app logo and name
- Email input field
- "Send me a link" button — triggers Supabase magic link
- Subtitle: "A one-time link will be sent to your email"

### 2. Lists

- **Header**: "הרשימות שלי" (My Lists) + overflow menu (⋮)
- **List cards**: emoji icon, list name, item count + checked count, last activity (dynamic — derived from `MAX(items.updated_at)`), chevron
- **New list**: subtle dashed row at bottom — secondary action
- **Bottom nav**: Lists (active) | Settings
- Tap a card → navigates to List Detail

### 3. List Detail

- **Top bar** (single row): `‹ back` · list title with emoji · `[🔍 search]` · `[+]` · `⋮`
  - Search bar is subtle, blends into header (no prominent border)
  - Search (🔍) filters/highlights items already on the current list
  - Plus (+) opens the Rapid Add canvas (screen 4)
- **Items**: grouped by category with section headers
  - Each row: unchecked box · item name · quantity pill (if > 1) · SVG icons for note/image indicators · right-aligned
  - Quantity pill: `x2`, `x3` etc. — only shown when quantity > 1
  - Note indicator: pen SVG icon (subtle, low opacity)
  - Image indicator: image SVG icon (subtle, low opacity)
- **Swipe left (←)**: reveals delete (🗑) first, then check (✓) — destructive action requires longer swipe
- **Swipe right (→)**: opens Item Detail (screen 5)
- **Tap checkbox**: quick check/uncheck
- **Checked items**: sink to bottom under "✓ סומנו" section, faded + strikethrough
- **Bottom nav**: Lists | Settings

### 4. Rapid Add Canvas (full screen)

Opened by tapping the `+` button on List Detail. Designed for speed — like writing a list in a notes app.

- **Header**: "הוספת פריטים" (Adding Items) + "סיום" (Done) button
- **Canvas**: blank area, type an item name, hit Enter
  - Item appears above with unchecked box
  - Cursor moves to next line — keyboard stays open
  - Type, Enter, type, Enter — rapid fire
- **Each added item row**:
  - Unchecked box · item name · quantity pill · (flex spacer) · category pill (right-aligned)
  - **Quantity = 1**: muted `+` pill (same style/color as category pill, same height) — tap to expand
  - **Quantity > 1**: active `x2` pill — tap to expand
  - **Expanded quantity**: full-width bar below the item row with generous `[ −   3   + ]` controls. Big touch targets.
  - **Category pill**: defaults to "אחר" (misc) for new items, auto-filled for known items from history. Tap to expand full-width category picker below (chip cloud of all categories). Never opens automatically — only on user intent.
- **Suggestion chips**: horizontal scrollable row above the keyboard
  - Sourced from group item history, matched by what you're typing
  - Tap a chip → auto-fills the line with that item + its remembered category
  - Passive — never blocks typing, not a dropdown
- **Done**: tap "סיום" → returns to List Detail with all new items added

### 5. Item Detail (full screen)

Opened by swiping right on an item. Closed by swiping back (right edge, iOS-style) or tapping back chevron.

- **Header**:
  - Top-left: avatar (profile image if set, otherwise first letter initial on colored circle) + timestamp (e.g., "14:32")
  - Center: item title
  - Top-right: back chevron (›) — RTL, points right to indicate "go back"
- **Content** (scrollable):
  - **Quantity**: stepper with −/+ buttons, large number in center
  - **Unit**: dropdown (יח׳, ק״ג, ליטר, etc.)
  - **Category**: combo field — type to search or tap dropdown chevron to pick from predefined list. Hint: "הקלד או בחר מהרשימה"
  - **Note**: free-text field for description/instructions
  - **Images**: at the bottom, full-width, scroll down to see. Stacked vertically. "הוסף תמונה" (Add image) dashed button at end. Camera + gallery capture.
- **Meta**: avatar + timestamp in header shows who added and when

## Design Principles

- **Mobile-first**: designed for use while walking around a store
- **Large touch targets**: 48px minimum tap areas
- **Speed over completeness**: rapid-add flow prioritizes speed. Category and quantity are optional tweaks, not required steps.
- **Be generous when there's intent**: when user taps a control (quantity, category), give them a full-width generous UI to interact with. Don't make them aim for tiny targets.
- **SVG icons only**: no emojis in UI chrome (list emojis are user-chosen content, not UI)
- **Light theme default**: warm, clean aesthetic inspired by Apple Notes. Light background with subtle grays. Follows system preference for dark mode support, but light is the primary design. Build with light theme from the start — retrofitting later changes every component.
- **System font stack**: no custom font loading

## Code Architecture

- Functional components with hooks
- Custom hooks for Supabase subscriptions (`useRealtimeItems`, `useRealtimeLists`)
- Supabase client in `lib/supabase.ts`
- Types in `types/` directory
- All Supabase queries through a `data/` layer, not directly in components
- i18n locale files in `locales/` directory
- Components kept small and focused

## What's Deferred (Post-MVP)

- Group settings / member management screen
- Full offline editing with change queue and conflict resolution
- Pull-to-refresh
- Haptic feedback on check
- Loading skeletons
- Custom app icon
- "Clear checked" bulk action
